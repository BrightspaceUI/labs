Basic Usage Demo

This demo component shows a very basic (and contrived) example of how the controller may be used. To get a better idea of what functionality and options this controller supports, take a look at the demo-component source code.
  • The fullName computed value is dependant on the firstName and lastName
  • The screamingFullName computed value is dependant on the fullName computed value
  • The asyncShortName computed value shows how a computed value can be async and that it can have custom shouldCompute logic
  • The altFullName computed value shows that you can use a single computed value instead of a collection of them

Class List Demo

This demo shows how the same UI can be made with or without the use of the ComputedValues controller. The resulting UI is identical, but check out the code for the <without-computed-values-component> and the <with-computed-values-component> to see the differences.

In this case, the UI fetches a list of students based on the selected class. Then it runs some front-end calculations to get the average grades and paginates the students.
